On the Subject of Sorting

What's the point of implementing a sorting algorithm if you are going to do it manually anyway?

This module has a screen at the bottom stating an algorithm, among 5 buttons above the screen laid out in an offset layout, each with a 2-digit label.

The defuser must be aware that positions refer to appearances from left to right, ignoring vertical offset. For example, 1st position classifies as bottom left.

Each label can be swapped by pressing any button - which will highlight it white - and then selecting any other button. Pressing the highlighted button during this state will cancel the swap. To disarm this module, swap the labels according to the algorithm until all labels have been sorted. If an invalid swap has been made, the module strikes and it resets back to the original positions.

Cycle Sort

Section 1
If... Then...
1st position isn't smallest? Refer to Section 2 with S = 1.
2nd position isn't 2nd smallest? Refer to Section 2 with S = 2.
3rd position isn't 3rd smallest? Refer to Section 2 with S = 3.
4th position isn't 4th smallest? Refer to Section 2 with S = 4.
Section 2
Find... Then...
The (S)th position. Count how many numbers are bigger than
(S)th position, store that value in B.
Swap (S)th position with (B + 1).
Refer to Section 1 unless sorted.

Quick Sort

Section 1
If... Then...
This is the first time running Section 1? Pivot = 1 and Current = 5.
Pivot is greater than current? Refer to Section 2.
Pivot's label is
greater than the current's label*?		 Swap pivot's label and current's label.
The placements of pivot and current themselves have also been switched**.
Refer to Section 3.
Section 2
If... Then...
Pivot's label is
smaller than the current's label*?		 Swap pivot's label and current's label.
The placements of pivot and current themselves have also been switched**.
Section 3
If... Then... Otherwise...
Pivot is smaller
than the current?		 Subtract 1 from current. Add 1 to current.
Pivot is the same as current? Pivot = earliest unsorted position.
Current = latest unsorted position.		 Nothing.
Refer to Section 1 unless sorted.

*Pivot's label and current's label refer to the label present on its position.
For example, if pivot = 2, then the pivot's label is the 2nd position's label.

**When any swap occurs, the pivot and current get swapped with each other as well. For example, if pivot = 1 and current = 5 and you have swapped those two labels, then afterwards pivot = 5 and current = 1.
